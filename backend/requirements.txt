# onnxruntime + onnxruntime-directml share the same `onnxruntime/` package dir;
# whichever was installed last wins at import time. openocr-python (below)
# hard-depends on plain `onnxruntime`, so a fresh `pip install -r` ends up with
# plain `onnxruntime` shadowing directml. To restore DirectML provider visibility
# on Windows, run after install:
#     pip install --force-reinstall --no-deps onnxruntime-directml
# `backend/build_app.py` performs this automatically before PyInstaller-freeze.
# The shadow pair must also stay VERSION-COHERENT: directml releases lag the
# plain package, and a newer plain `__init__` over the older directml capi
# fails at import (`OrtDeviceVendorId`, plain 1.26.0 over directml 1.24.4), so
# the Windows shadow pins to the directml release line.
